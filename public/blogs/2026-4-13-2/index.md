### 二分搜索算法核心代码模板

二分查找常被视作入门级算法，但其实际落地却极具欺骗性。就连算法先驱 Donald Knuth 也曾坦言，尽管其核心思想一目了然，但工程实现却处处暗藏玄机。不少初学者将注意力过度集中在“防止 mid 计算溢出”这类语法规范上，却忽略了真正的难点：循环终止条件究竟该用 < 还是 <=？区间收缩时到底该让指针越过 mid 还是保留 mid？哪怕只是一个索引的偏差，都可能引发死循环或漏判目标值。

若缺乏清晰的区间逻辑模型，编写二分查找往往会陷入“试错式编程”的泥潭——改一个符号跑一次测试，能否通过全凭运气。为了彻底摆脱这种盲目调试的困境，本文将系统梳理二分查找的三大高频应用场景：精准定位目标值、查找目标值的左边界（首次出现位置）、以及查找右边界（末次出现位置）。我们将采用一套高度统一的代码骨架：始终维持闭区间搜索逻辑（**while (left <= right)**），并固定使用 mid ± 1 来安全收缩范围。掌握这一标准化范式后，你便能将原本脆弱易错的逻辑，转化为稳定可复用的解题模板。

```cpp
class Solution {
public:
    int search(vector<int>& nums, int target) {
        
        int left = 0;
        int right = nums.size() - 1;
        while(left <= right){
            int mid = (left + right)/2;

            if(nums[mid] == target)
                return mid;
            else if (nums[mid] <= target)
                left = mid + 1;
            else if (nums[mid] >= target)
                right = mid - 1;
            
        }
            return  -1;
    }
};
```
